Eine statische Website ohne Generator

WebTutorial

Statische Seiten-Generatoren wie Hugo oder Jekyll sind mächtig — aber für einen kleinen Blog können sie sich anfühlen wie ein Presslufthammer für einen Reisnagel. Diese Seite kommt ganz ohne aus.

Das Grundprinzip

Es gibt genau drei bewegliche Teile:

  • eine Startseite (index.html) mit einer Liste der Beiträge,
  • pro Beitrag eine eigene HTML-Datei im Ordner posts/,
  • ein gemeinsames Stylesheet (assets/blog.css) für das einheitliche Aussehen.

Wie Filter und Tags funktionieren

Die Startseite enthält eine kleine Liste meiner Beiträge als JavaScript-Objekt — mit Titel, Datum, Tags und einer kurzen Vorschau:

const posts = [
  {
    title: "Mein Beitrag",
    date: "2026-05-28",
    url: "posts/2026-05-28-mein-beitrag.html",
    tags: ["Web", "Tutorial"],
    excerpt: "Worum es geht."
  }
];

Aus dieser Liste baut der Browser die Beitragskarten, sammelt automatisch alle Tags ein und filtert beim Tippen oder Tag-Klick. Kein Server rechnet etwas aus — alles passiert lokal im Browser des Besuchers.

Ein neuer Beitrag = eine neue HTML-Datei plus ein neues Objekt in der Liste. Das ist der ganze Arbeitsablauf.

Warum mir das reicht

Ich tausche etwas Komfort (automatische Übersichten, Markdown) gegen volle Kontrolle und null Abhängigkeiten. Die Seite wird vermutlich noch laufen, wenn das nächste Generator-Major-Release längst alles umgekrempelt hat.
